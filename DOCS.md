# File MIME Checker 프로젝트 문서화 및 학습 정리

## 1. 개요 및 프로젝트 목표
본 프로젝트는 단일 애플리케이션 안에서 파일의 MIME 타입을 확인하는 기능을 제공하며, 사용 환경에 따라 터미널 직접 실행(CLI) 및 웹 서비스(REST API) 모드를 모두 지원하는 다목적 유틸리티임
- 객체지향 설계(디자인 패턴 적용)
- 단일 코드베이스의 다중 진입점 분기

## 2. 도출된 기능 리스트
- 실행 모드 스위칭: 메인 애플리케이션 실행 시 -d 옵션의 유무를 파악. 서버 모드(Spring Boot) 또는 단발성 CLI 모드로 자동 분기 처리.
- MIME 타입 감지 로직:  Apache Tika를 활용한 빠르고 정확한 파일 타입 분석.
- ZIP 파일 압축 해제 및 탐지:
- REST API 제공: 서버 모드 실행 시 /upload 엔드포인트를 통해 파일을 업로드받고 결과를 문자열로 반환.

## 3. 적용된 설계 및 디자인 패턴
본 프로그램의 비즈니스 로직 확장성과 유지보수성을 극대화하기 위해 다음 두 가지 패턴을 핵심적으로 도입하였습니다.

### 가. 전략 패턴 (Strategy Pattern)
- 적용 목적: MIME 감지 알고리즘을 유연하게 교체하거나 추가할 수 있도록 분리하기 위함.
- 구현 방식: MimeDetectionStrategy 인터페이스를 공통으로 정의하고, TikaMimeDetectionStrategy 구현체를 사용하도록 의존성을 주입받아 사용
- 기대 효과: 새로운 감지 로직이 요구될 때, 기존 코어(FileMimeCore)의 코드를 수정하지 않고 전략 클래스만 추가해 소스 확장 가능

### 나. 빌더 패턴 (Builder Pattern)
- 적용 목적: 여러 개의 파일(특히 압축 파일 내부의 파일들)에 대한 판별 결과를 안전하고 직관적으로 수집 및 생성하기 위함.
- 구현 방식

  - 결과 데이터를 담는 MimeDetectionResult 클래스 내부에 정적 Builder 클래스를 둠
  - 결과를 지속적으로 추가(addMimeType)
  - build() 메서드를 호출하여 불변(Immutable) 상태의 최종 객체를 완성합니다.
- 기대 효과: 데이터 객체의 생성 과정을 캡슐화하여 코드의 가독성을 대폭 향상시키고, 생성 중간 단계에서의 데이터 오염을 방지하여 안전성을 확보합니다.

## 4. 결론 및 느낀 점
하나의 프로젝트 안에 Spring Boot의 웹 프레임워크 기능과 순수 Java 콘솔 기반의 CLI 구동 기능을 충돌 없이 매끄럽게 통합하는 방법을 확인했습니다.
전략 패턴과 빌더 패턴을 통하여 코어 로직을 수정하지 않고 확장 가능한 객체 지향 프로그래밍의 기초를 익혔습니다.
